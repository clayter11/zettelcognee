# n8n-nodes-scenario-parser

Это кастомная нода для n8n, которая парсит текст сценариев и структурирует их в формате "Персонаж - Реплика", а затем сохраняет результат в Google Docs.

## Возможности

- **Единая нода** - все функции инкорпорированы в одну ноду без внешних зависимостей
- Парсинг сценариев в различных форматах
- Структурирование данных в читаемом формате
- Автоматическое сохранение в Google Docs через Service Account
- Поддержка различных форматов кавычек (« », " ", „ ")
- Автоматическая нумерация реплик
- Добавление временных меток
- Возможность очистки документа перед записью

## Установка

1. Установите зависимости:
```bash
npm install
```

2. Соберите проект:
```bash
npm run build
```

3. Установите ноду в n8n:
```bash
npm link
```

## Быстрый старт

1. **Создайте Google Service Account** и скачайте JSON ключ
2. **Создайте Google документ** и дайте доступ Service Account
3. **Добавьте ноду** в ваш n8n workflow
4. **Заполните поля**:
   - Scenario Text: ваш текст сценария
   - Google Service Account Key: JSON ключ
   - Google Document ID: ID документа из URL

## Поддерживаемые форматы входных данных

### Полный формат (рекомендуется):
```
Счастливая сторона, Счастливая Маша, «Я так хочу поделиться своими идеями!»
Критикующий режим, Вреднюга, «Ты глупа и не можешь сказать ничего ценного!»
```

### Упрощенный формат:
```
Счастливая Маша, «Я так хочу поделиться своими идеями!»
Вреднюга, «Ты глупа и не можешь сказать ничего ценного!»
```

### Без кавычек:
```
Счастливая Маша, Я так хочу поделиться своими идеями!
Вреднюга, Ты глупа и не можешь сказать ничего ценного!
```

## Выходной формат в Google Docs

```
Структурированный сценарий (28.07.2024, 10:30)

1. Сторона: Счастливая сторона
   Персонаж: Счастливая Маша
   Реплика: "Я так хочу поделиться своими идеями!"

2. Сторона: Критикующий режим
   Персонаж: Вреднюга
   Реплика: "Ты глупа и не можешь сказать ничего ценного!"

--- Итого обработано реплик: 2 ---
```

## Настройки ноды

- **Scenario Text** - текст сценария для парсинга (обязательно)
- **Google Service Account Key** - JSON ключ Service Account (обязательно)
- **Google Document ID** - ID Google документа (обязательно)
- **Clear Document Before Writing** - очистить документ перед записью (по умолчанию: true)
- **Add Timestamp** - добавить временную метку в заголовок (по умолчанию: true)

## Требования

- n8n версии 0.174.0 или выше
- Google Cloud Project с включенным Google Docs API
- Google Service Account с JSON ключом
- Google документ с доступом для Service Account
